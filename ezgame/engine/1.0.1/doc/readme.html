EasyGame前端引擎使用说明

引擎安装和开发环境的准备

  1. 安装nodejs
  2. 安装ezgame包，在命令行环境运行npm i ezgame -g
  3. 安装vscode
  4. 在vscode中安装ezgame插件

新建一个游戏项目

  • 为游戏项目创建一个新的文件夹
  • 用vscode打开这个文件夹
  • 打开vscode的终端界面
  • 在该文件夹下运行ez-cli init
  • 输入一些项目相关的信息之后就初始化好了项目，生成后的项目目录结构如下：
project
  |---assets
        |---res           资源文件存放目录
        |---resource.json 资源清单文件
  |---engine              EasyGame引擎文件
  |---types               EasyGame引擎的ts类型声明文件
  |---src                 存放项目源代码
  |---ui                  存放ui模板文件
  |---index.html          主页面
  |---ez.config.hjson     项目配置文件
  |---tsconfig.json       ts项目文件
  • 运行ez-cli scan来扫描资源清单
  • 使用ez-cli build构建游戏
  • 使用ez-cli run -w运行游戏，现在就可以使用浏览器打开 http://localhost:8001/ 来查看游戏了
  • ez-cli run的-w开关会让引擎在后台自动检查项目文件的变化并重新构建游戏，当构建完成后前端游戏页面会得到更新通知，提醒你刷新页面查看变更后的效果。这意味着用户每次修改完代码或者资源文件，按下CTRL+S的时候引擎就自动帮你完成了构建工作，之后就可以立刻刷新页面看到修改后的结果了。

ez-cli命令行说明

ez-cli是EasyGame引擎的命令行工具，可以使用这个工具完成项目的创建、构建、运行、发布以及升级等操作。

基本命令：
  ez-cli init                               在当前目录中创建一个新工程

  ez-cli scan [config]                      扫描资源文件目录并更新资源文件清单

  ez-cli build [config]                     构建项目

  ez-cli publish [config] [--jsonly]        将资源文件发布到分发目录中，并将资源描述清单嵌入到ts代码中

  ez-cli run [config] [-u host] [-p port]   在浏览器中运行项目

  ez-cli upgrade [version]                  更新EasyGame引擎版本

详细的命令用法可以使用 ez-cli command -h 来查看

ez.config.hjson配置说明

ez.config.hjson是项目的配置文件，在这个配置文件里你可以为开发环境和各种发布环境配置不同的参数。配置文件使用Hjson格式来描述。在生成的配置模板中已经为各个配置参数添加了注释说明。

资源文件管理

游戏用到的所有资源文件都存放在assets/res目录下，你可以创建更多的子目录以便对资源文件进行分类存取。在游戏中使用资源文件的相对路径来引用资源文件（去除资源文件的扩展名），例如assets/res/ui/btn.png这个资源在游戏中使用ui/btn来引用。每次添加、删除资源文件时都需要重新扫描资源清单。

扫描清单时，引擎根据资源文件的扩展名来区分资源的类型，目前支持的扩展名类型有：

扩展名 类型
png/jpg 图片资源
mp3 音乐/音效
txt 文本
json JSON数据
csv 表格数据
gltf gltf 3D模型
binary 二进制数据
spine spine动画

利用vscode的EasyGame扩展插件，我们可以给图片资源添加一些附加属性，例如：9宫格拉伸信息、图像大小、空白边距、纹理格式等等信息。操作方法是在图片资源上点击右键菜单，选择添加属性。此时就会创建一个和图片同名的.meta文件，使用JSON格式添加需要的属性。

利用扩展插件我们还可以把一些小图片自动合成一张大的纹理图册(atlas)，减少纹理数量优化渲染性能。首先将需要合并的图片整理并放到同一目录下，图像合并将以目录为单位，将这个目录下所有的图片合并到一起，右键点击目录选择图片合成atlas，将显示这个界面:

调整好参数后点击开始按钮就完成了图片的合并。此时会生成和目录同名的png文件和meta文件，meta文件中包含了每个子图片的atlas信息。原始的图片和目录建议保留，方便日后进行增删改，每次修改后再次运行图片合成atlas命令重新合并即可。原始图片不会被扫描到资源清单文件里，也不会被最终发布。

ui模板文件

EasyGame使用uiml格式来定义UI模板，在ui模板中定义每个ui的样式、布局、状态以及数据绑定，从而实现了ui样式和ui逻辑代码的分离。uiml的语法基本来自xml，但是比xml更为宽松，属性值可以不用添加引号，bool类型的属性可以不需要值（默认为true），以此生成更为简洁清晰的模板。最终所有的ui模板将被编译转换成ui.ts文件添加到工程中。EasyGame扩展插件为uiml实现了语法高亮和输入提示。

一个Button的例子：

<?xml version="1.0" encoding="utf-8" ?>
<root module=game>
    <Button baseclass=Control width=194 height=70>
        <properties>
            <e name=label type=string />
        </properties>
        <styles>
            <blue>
                <bk src=ui/btn/blue />
            </blue>
            <purple>
                <bk src=ui/btn/purple />
            </purple>
            <yellow>
                <bk src=ui/btn/yellow />
                <label gradient="{ y0:0, y1:30, colors: ['#a54800', '#ce7300'] }" />
            </yellow>
        </styles>
        <children>
            <Image id=bk src=ui/btn/blue width=100% height=100% />
            <Label id=label font=30px color=#fff width=100% height=30 y=50% align=Center />
        </children>
        <dataBinder>
            <e label.text=this.label />
        </dataBinder>
        <states default=normal>
            <normal/>
            <down/>
        </states>
        <eventHandler args=0.8>ui.addButtonEventHandler</eventHandler>
    </Button>
</root>

游戏的开发调试

在开发环境中，EasyGame在游戏页面里提供了一个inspector面板，方便用户查看和调试游戏。

在Profiling页签中可以查看游戏的当前的性能数据，并且可以通过capture命令捕获当前帧的绘制命令列表，从而指导用户进行优化。

在UI页签中可以查看当前游戏的ui对象树，这相当于HTML中的DOM树。用户可以查看每个UI元素的当前位置和属性参数，并对属性进行修改，观察UI元素的变化。

在Test页签中可以运行用户编写的各种测试用例，方便快速的进行游戏逻辑的开发调试。合理的编写测试用例可以让你快速的验证和调试游戏中的各个子模块，提高开发效率。EasyGame引擎的模块化设计也让各个模块能够被独立测试提供了可能性。

游戏的发布

EasyGame支持发布到各种平台上，目前支持的平台有：

  • 各大浏览器平台 (chrome, firefox, safari, edge, IE等等)
  • 微信内置浏览器
  • 微信小游戏
  • QQ小游戏
  • 基于WebView的Andriod原生应用
  • 基于WebView的iOS原生应用
  • windows原生应用 -- 开发中
  • Android原生应用 -- 开发中
  • iOS原生应用 -- 开发中

发布时在ez.config.hjson中添加平台的配置参数，设置对应的PUBLISH类型和发布目录。然后运行ez-cli publish [cfg]，引擎就开始对资源文件进行hash处理，资源文件名会替换成文件内容的hash值，并写入到资源清单文件中，资源清单文件也会转换成res.ts文件，嵌入到游戏工程中。因为使用了文件内容的hash值作为资源文件名，因此玩家在下载游戏资源时总能获得最新版本的资源文件，而不用担心文件缓存的刷新。

完成资源文件的发布后引擎会继续对游戏js代码进行合并、压缩、混淆，并输出最终版本的js和html文件。
